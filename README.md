# 🌱 KrishiMitraaz — Smart Crop Advisory (SIH25010)

KrishiMitraaz is a fully-featured, full-stack **Web Dashboard** and **API Engine** designed to provide voice-ready, weather-aware, and highly grounded agricultural advisory for small and marginal farmers. 

> **Philosophy**: Advice must be **grounded and explainable**, not free-form LLM guesses. Every advisory item generated by our backend engine carries a `basis` (its reasoning trace) so it can be spoken aloud with a "why". That is the trust story judges reward.

---

## 🌟 Unique Value Propositions (UVP)
- **Professional Government Dashboard:** A highly accessible, solid-theme enterprise UI (NIC-style) designed for clarity and authority.
- **Precision Stage Tracking:** Registration flow captures Sowing Date and Land Area to provide day-0 personalized crop stage advisory and fertilizer scaling.
- **Farmer Profile & Multi-Year Yield History:** Dynamic frontend profile menu with integrated Real-Time Yield APIs. Farmers can track multiple years of harvest data directly from the portal.
- **Voice-Ready Summaries:** The AI Advisory Engine generates perfectly structured, prioritized text designed specifically for Bhashini Text-to-Speech parsing.
- **Microclimate Crop Suitability:** AI-driven crop suggestions based entirely on live location-based weather data.
- **Real-Time Financial Projections:** A Yield Calculator that integrates real-time MSP, seed, and fertilizer costs to project **Profit Margins** per acre.
- **Precision Geolocation:** Integrates `navigator.geolocation` and **Nominatim Reverse Geocoding** to seamlessly detect a farmer's village and state.

## 🚀 Quick Start (Zero Setup — Runs Locally!)

We've designed this repository so judges can run the entire platform locally with zero external database setup required. The backend uses a lightweight JSON-file repository (`store.json`), and the frontend is a blazing-fast React Vite app.

### 1. Start the Backend API

```bash
cd krishimitraaz
npm install

# Seed the local JSON database with demo farmers
npm run seed      

# Start the API on http://localhost:3000
npm start         
```

### 2. Start the Frontend Dashboard

Open a **second terminal** window:

```bash
cd krishimitraaz/frontend
npm install

# Start the React Vite Dashboard
npm run dev
```

Visit the local URL provided by Vite (usually `http://localhost:5173`). 
Log in securely using an Official ID!

*(Note: To create a production build for the frontend, simply run `npm run build` in the `frontend` folder.)*

## 🛠️ Environment Configuration (Production)

We follow strict security standards. Hardcoded URLs are avoided.

### Frontend
To deploy the frontend to production, create a `.env` file in the `frontend/` directory (you can copy `.env.example`):
```env
VITE_API_URL=https://api.yourdomain.com/api
```

### Backend
To hook the backend up to real data providers instead of mock data, copy `.env.example` to `.env` in the root directory:
```env
WEATHER_PROVIDER=openweather
OPENWEATHER_API_KEY=your_key

MARKET_PROVIDER=agmarknet
DATAGOV_API_KEY=your_key
```
*Nothing else changes — our internal API callers depend only on the normalized data shapes.*

## 🏗️ Technical Architecture

- **Frontend:** React + Vite (HTML/JS/CSS)
- **Backend:** Node.js + Express
- **State Management:** Secure JWT Tokens
- **Databases (Future Scale):** PostgreSQL + PostGIS (for geospatial), MongoDB (for chatbot logs)
- **APIs Used:** ISRO Bhuvan (NDVI), IMD Weather APIs, OpenStreetMap Nominatim, Agmarknet Market Prices
